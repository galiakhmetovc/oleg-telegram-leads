# Config V3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the evolved v2 NLP semantic configuration with a clean v3 model: dictionaries enrich text into facts, facts produce domain signals, and lead scoring requires domain evidence plus explicit intent.

**Architecture:** Keep the existing FastAPI/PostgreSQL/settings/Yargy execution mechanism, but replace the semantic taxonomy and default YAML documents. The new model has four explicit layers: alias catalogs, fact types, domain signals, and lead assessment taxonomy. Active PostgreSQL config is patched by Alembic because PostgreSQL revisions are the editable source of truth.

**Tech Stack:** Python 3.12, FastAPI, SQLAlchemy/Alembic, Natasha/Yargy, PostgreSQL JSONB config revisions, React/Vite/MUI settings UI.

---

## File Structure

- Modify `backend/config/nlp/facts.yaml`: replace old mixed facts with v3 facts grouped by intent, project context, object context, work context, and domain evidence.
- Modify `backend/config/nlp/signals.yaml`: replace old broad/cross-domain signals with v3 signals that depend on facts and aliases through `match.facts`.
- Modify `backend/config/nlp/lead_scoring.yaml`: replace old scoring weights/areas/segments/review lanes with v3 weights, lead caps, and lanes.
- Modify alias catalogs only where a dictionary item emits an overly broad fact type.
- Add Alembic migration in `backend/alembic/versions/`: patch active PostgreSQL config to v3 documents.
- Modify `backend/tests/test_enrichment_pipeline.py`: replace old default-config expectations with v3 behavior and golden examples.
- Modify `backend/tests/test_lead_scorer.py` only if scorer mechanics need a new generic behavior test.
- Update `docs/architecture.md`, `docs/decisions.md`, `state/current.md`, and `frontend/src/settings/SettingsHelpPage.tsx`.
